Class: SubcellLayout

SubcellLayout

Layout of a subcell, hardcoded to 3 subviews (main, sphere and slicer) for now

Constructor

new SubcellLayout(args)

Constructs a new subcell layout
Parameters:
Name Type Description
args Object Multipliers to scale the inner layout. The inner layout has 2 multipliers: SnapThreshold, and standardSize. (NOTE: Maybe make it more dynamic to add more widgets, for now only 2 subcells are hardcoded in here.)
Properties
Name Type Description
snapThresholdMultiplier number The threshold to snap the layout to a different scheme. Decides WHEN one of the following conditions arise:
(1) : width > snapThreshold * height:
will put the subviews on the right and have them use all the available height, hence the side length of the views will be height/2.

(2): height > snapThreshold * width
will put the subviews on the bottom and have them use all available width. Side length of subcells will be width/2

If none of the snap thresholds are met, the standardSize multiplier will decide the layouting
standardSizeMultiplier number Scales the size of the two subcells, which will be put in the lower right corner and have a side length of standardSize*(shortest cell side)/2
